# 特定应用的区块链
{: .highlight }
>本文档解释了什么是特定应用的区块链，以及为什么开发者会想构建一个这样的区块链，而不是编写智能合约。

## 什么是特定应用的区块链
特定应用的区块链是为运行单一应用而定制的区块链。开发者不是在像以太坊这样的基础区块链之上构建去中心化应用，而是从头开始构建自己的区块链。这意味着要构建一个全节点客户端、一个轻客户端以及所有必要的接口（CLI、REST 等）来与节点进行交互。

## 智能合约的缺点
像以太坊这样的虚拟机区块链在2014年回应了对更多可编程性的需求。当时，构建去中心化应用的选择相当有限。大多数开发者会在复杂且受限的比特币脚本语言之上构建，或者分叉难以使用和自定义的比特币代码库。

虚拟机区块链带来了新的价值主张。它们的状态机内置了一个能够解释图灵完备程序（称为智能合约）的虚拟机。这些智能合约非常适合像一次性事件（例如 ICO）这样的用例，但在构建复杂的去中心化平台时可能会不足。原因如下：

* 智能合约通常是用特定的编程语言开发的，这些编程语言可以被底层虚拟机解释。这些编程语言通常不成熟，且本质上受到虚拟机本身的限制。例如，以太坊虚拟机不允许开发者实现代码的自动执行。开发者还受限于 EVM 的基于账户的系统，他们只能从有限的加密操作功能集中选择。这些只是例子，但它们暗示了智能合约环境通常涉及的灵活性缺乏。
* 智能合约都由同一个虚拟机运行。这意味着它们竞争资源，可能严重限制性能。即使状态机被分割成多个子集（例如通过分片），智能合约仍然需要通过虚拟机解释，与在状态机级别实现的原生应用相比，性能会受到限制（我们的基准测试显示，在去除虚拟机时性能提升了约10倍）。
* 智能合约共享相同底层环境的另一个问题是主权的限制。去中心化应用是一个涉及多个参与者的生态系统。如果应用是构建在通用虚拟机区块链上的，利益相关者对他们的应用几乎没有主权，并最终受制于底层区块链的治理。如果应用中存在漏洞，几乎无法采取措施。

特定应用的区块链旨在解决这些缺点。

## 特定应用的区块链优势
### 灵活性
特定应用的区块链为开发者提供了最大的灵活性：

* 在 Cosmos 区块链中，状态机通常通过一个称为 ABCI 的接口连接到底层共识引擎。这个接口可以用任何编程语言包装，这意味着开发者可以用他们选择的编程语言来构建他们的状态机。

* 开发者可以在多个框架中选择来构建他们的状态机。目前最广泛使用的是 Cosmos SDK，但也存在其他选择（例如 Lotion, Weave 等）。通常，选择将基于他们想要使用的编程语言（Cosmos SDK 和 Weave 是 Golang，Lotion 是 Javascript 等）。

* ABCI 还允许开发者更换他们特定应用区块链的共识引擎。目前，只有 CometBFT 是生产就绪的，但未来预计会出现其他共识引擎。

* 即使他们确定了框架和共识引擎，开发者仍然有自由去调整它们，如果它们原始形式不完全符合他们的要求。

* 开发者可以自由探索各种权衡（例如验证者数量与交易吞吐量、异步中的安全性与可用性等）和设计选择（DB 或 IAVL 树用于存储，UTXO 或账户模型等）。

* 开发者可以实现代码的自动执行。在 Cosmos SDK 中，逻辑可以在每个区块的开始和结束时自动触发。他们还可以自由选择应用中使用的加密库，而不像虚拟机区块链那样受限于底层环境提供的内容。

上面的列表包含了一些例子，展示了特定应用的区块链给开发者提供了多少灵活性。Cosmos 和 Cosmos SDK 的目标是使开发者工具尽可能通用和可组合，以便每个堆栈部分都可以被分叉、调整和改进，同时不失去兼容性。随着社区的增长，核心构建块的更多替代品将出现，给开发者提供更多选择。

### 性能
使用智能合约构建的去中心化应用在性能上天生受限于底层环境。要优化去中心化应用的性能，需要将其构建为特定应用的区块链。以下是特定应用区块链在性能方面带来的一些好处：

* 特定应用区块链的开发者可以选择使用像 CometBFT BFT 这样的新型共识引擎。与今天大多数虚拟机区块链使用的工作证明（Proof-of-Work）相比，它在吞吐量上提供了显著的增益。
* 特定应用的区块链只操作单一应用，因此应用不需要与其他应用竞争计算和存储资源。这与今天大多数非分片虚拟机区块链恰恰相反，其中智能合约都在竞争计算和存储资源。
* 即使虚拟机区块链提供了基于应用的分片以及高效的共识算法，性能仍然会受到虚拟机本身的限制。真正的吞吐量瓶颈是状态机，而且要求交易被虚拟机解释会显著增加处理它们的计算复杂性。

### 安全性
安全性难以量化，且因平台而异。尽管如此，以下是

特定应用区块链在安全性方面可以带来的一些重要优势：

* 开发者可以在构建特定应用区块链时选择经过验证的编程语言，如 Go，而不是经常更不成熟的智能合约编程语言。
*开发者不受限于底层虚拟机提供的加密函数。他们可以使用自己的定制加密，并依赖于经过良好审核的加密库。
*开发者不必担心底层虚拟机中可能存在的漏洞或可利用机制，这使得对应用的安全性更容易推理。

### 主权
特定应用区块链的一个主要优势是主权。去中心化应用是一个涉及许多参与者的生态系统：用户、开发者、第三方服务等。当开发者在许多去中心化应用共存的虚拟机区块链上构建时，应用的社区与底层区块链的社区不同，后者在治理过程中优先于前者。如果存在漏洞或需要新功能，应用的利益相关者几乎没有余地升级代码。如果底层区块链的社区拒绝采取行动，什么也不会发生。

这里的根本问题是应用的治理与网络的治理不一致。特定应用的区块链解决了这个问题。因为特定应用的区块链专门用于操作单一应用，应用的利益相关者对整个链拥有完全控制权。这确保了社区在发现漏洞时不会受困，并且有自由选择它将如何发展。
